Task Manager 06 - Relational Database

This application demonstrates a normalized relational database design with:

  • Users - People who create and are assigned tasks
  • Tasks - Todo items with status and details
  • Tags - Categories for organizing tasks

Database Relationships

  • User → Tasks: One-to-many (one user can have many tasks)
  • Tasks ↔ Tags: Many-to-many (tasks can have multiple tags, tags can be on multiple tasks)
